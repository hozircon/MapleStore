# 需求規格文件：輕量化交易展示站 (Lightweight Trading Showcase)

## 1. 專案背景與目標

本專案旨在解決舊版本楓之谷（私服）玩家在社群（如 Line, Discord）或論壇中，僅能透過低效率截圖展示物品的問題。

**期望 1 (核心)**：建立一個具備搜尋與篩選功能的商品展示頁，方便管理員發布個人庫存，讓買家能快速定位所需商品。

**期望 2 (擴充)**：未來可作為代上架平台，向其他玩家收取虛擬貨幣手續費以提供展示服務。

## 2. 系統架構建議

- **後端**: Java (Spring Boot) + Spring Data JPA
- **資料庫**: H2 / MySQL（建議將 XML 靜態資料轉入 DB 以利過濾搜尋）
- **前端**: Thymeleaf + Bootstrap 5（或 Tailwind CSS）
- **資源**: 介面圖示與道具數據取自遊戲 Client 端解包之 XML 檔案

## 3. 功能需求與 UI 設計

### 3.1 主頁面配置 (左右切分佈局)

**左側：搜尋與篩選欄 (佔比 20-25%)**

- **模糊搜尋**：關鍵字輸入框（搜尋商品名稱）
- **大類切換**：裝備、消耗、裝飾、其他
- **貨幣過濾**：Checkbox 選項（楓幣、CS、WS），預設全選
- **進階篩選 (依大類變動)**：
  - **裝備**：下拉選單選取「職業」、「部位」，以及數值輸入框（如：力量 >= ?、攻擊力 >= ?）
  - **消耗/其他**：下拉選單選取次分類（如：卷軸、材料、礦石）
- **搜尋按鈕**：手動觸發查詢

**右側：結果顯示區 (佔比 75-80%)**

- **商品列表**：區分為「裝備類」與「一般類」兩種顯示模板

### 3.2 商品顯示模板

**【裝備類模板】 (強調數值差異)**

- **基本資訊**：道具圖示 + 道具名稱
- **數值區**：採直列式顯示（STR, DEX, INT, LUK, ATK, MATK, 剩餘捲次）
- **價格區**：顯示貨幣圖示（楓幣/CS/WS）+ 數值
- **賣家資訊**：賣家名稱、所在頻道/自由市場洞數

**【一般類模板】 (如消耗、其他)**

- **欄位**：編號、圖示、名稱、數量、價格、賣家資訊

## 4. 資料邏輯設計

### 4.1 排序邏輯 (Priority Sorting)

為符合以物易物市場特性，搜尋結果需支援多貨幣優先級排序：

- **優先級 1**：標價為「楓幣」之商品
- **優先級 2**：標價為「CS (混沌卷軸)」之商品
- **優先級 3**：標價為「WS (祝福卷軸)」之商品

同一優先級內，依價格數值由小到大排序。

### 4.2 庫存更新機制 (管理員導向)

- **手動更新**：因無法串接 API，管理員需定期手動下架已售完商品
- **快速操作**：後台提供「一鍵下架」或「一鍵更新過期時間」功能，減輕管理壓力

## 5. 資料庫實體設計 (Entity Design)

| 欄位名 | 類型 | 說明 |
|---|---|---|
| `item_id` | Integer | 對應 XML 中的道具 ID |
| `category` | String | 大類 (裝備, 消耗, 等) |
| `sub_category` | String | 小類 (武器類型, 職業等) |
| `stats_json` | Text/JSON | 儲存裝備數值 (或拆分為獨立 Column 以利進階搜尋) |
| `price_type` | Enum | MESO, CS, WS |
| `price_value` | Long | 標價數值 |
| `quantity` | Integer | 庫存數量 |
| `location` | String | 遊戲內位置 (如: CH1 自由03) |
| `seller_name` | String | 賣家角色 ID |
| `status` | Boolean | 是否在庫 (In Stock / Sold Out) |

## 6. 非功能性需求

- **聯絡功能簡化**：不開發即時聊天，改為提供「複製交易暗號」按鈕，讓買家在遊戲內直接私訊賣家
- **未來擴充性**：
  - 保留「主打商品頁」佈局彈性（現代化左右大圖展示）
  - 資料結構預留 `featured` 標記位，用於未來收費推廣功能
